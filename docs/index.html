Heroku Platform MCP Server

An MCP server that exposes the Heroku platform as agent-callable tools — apps, dynos, Postgres, Key-Value, Kafka. Self-hosted, OAuth-protected, open source.

Powered by Salesforce Platform · Runs on Heroku · Speaks MCP

Get started

Two paths. Same MCP server. Pick the one that fits your situation.

1

Use a shared instance

Skip the setup. Point your MCP client at an existing deployed server and authenticate with your own Heroku account. Tools act on your behalf using your own permissions — you never share credentials with the host.

Jump to Connect from Claude →
2

Deploy your own

Run your own private instance on Heroku. The deploy form needs a Heroku OAuth client ID and secret — the credentials the server uses to call Heroku on your behalf. Create them with one CLI command, then click the button. The setup walkthrough covers the exact ordering.

What it does

276 tools spanning the full Heroku platform surface. Every tool callable by any MCP-aware agent.

Platform

Heroku Platform API

Apps, dynos, builds, releases, config vars, addons, teams, spaces, webhooks, OAuth clients.

Data

Heroku Postgres

Database info, credentials, backups, followers, maintenance. Plus write operations with confirm guards.

Data

Key-Value Store (Redis)

Admin, credentials, config (maxmemory / timeout / keyspace notifications), stats.

Streaming

Apache Kafka on Heroku

Cluster info, topics, consumer groups.

Two endpoints, pick your context budget

Standard

/mcp

Connect from Claude Desktop

Five steps. Works in claude.ai web the same way.

Using a shared instance? Use the URL you were given. Deployed your own? Your endpoint is https://YOUR-APP-NAME.herokuapp.com/mcp (or /mcp-codemode), where YOUR-APP-NAME is the app name you picked at deploy.

  1. Open Settings → Connectors → Add custom connector
  2. Paste either endpoint URL — your choice between /mcp and /mcp-codemode
  3. Leave the connector's OAuth Client ID and Secret blank — these are Claude's own registration fields, which the server fills automatically via Dynamic Client Registration. (This is unrelated to the Heroku OAuth client you created when deploying your own instance.)
  4. Click Add. A browser window opens for Heroku authentication. Sign in and click Authorize
  5. Connector shows as Connected. You now have access to your Heroku resources through Claude

Switching between endpoints

You can change endpoints at any time. Remove the current connector, add a new connector pointing at the other URL, complete a fresh OAuth flow. Old connections at /mcp and new connections at /mcp-codemode coexist — you're not locked in.

Other MCP clients

Most clients accept a connector URL plus OAuth configuration. The server speaks standard MCP over HTTP with Dynamic Client Registration, so any spec-compliant client works. Configuration syntax varies by client; the URL is the same. If your client requires explicit OAuth client_id/secret (no DCR support), see the OAuth setup guide linked below.
